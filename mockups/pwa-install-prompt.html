PWA install prompt — mockup

Three screens: normal portal header · Android spotlight · iOS spotlight. Behavior: a small "Install on phone" button sits in the portal header on every page. Tapping it fires the spotlight. Already-installed users never see it (we detect display-mode: standalone). One-time dismissal stored in localStorage so they don't get re-prompted on every visit.

Small button next to "My profile" — discoverable, not intrusive. Same affordance on instructor / parent / provider portals.

2. Android — tap fires native install

On Chromium browsers (Chrome, Edge, Samsung Internet), tapping our button calls the captured beforeinstallprompt event. The browser shows its own native sheet — we don't need to draw one. Our spotlight is just the brief affirmation card.

3. iOS — manual Share → Add to Home Screen

iOS Safari doesn't expose a programmatic install. We detect iOS user-agent + navigator.standalone === false and render a card with the two-step instructions and an annotated arrow pointing at the Share icon.

Behavior summary

  • Trigger: header "Install on phone" button (no auto-fire on first visit).
  • Hide automatically when app is already installed (display-mode: standalone).
  • iOS path: spotlight card with annotated Share-icon steps.
  • Android / Chrome path: native install sheet via beforeinstallprompt.
  • Service worker update (separate from install): when a new version deploys, a small toast appears at the bottom: "New version available — Tap to update." One tap reloads. Never auto-reloads mid-session.
  • Scope: manifest at /manifest.webmanifest, scope /, start URL /. Same install works for any portal (instructor / parent / provider / admin).
