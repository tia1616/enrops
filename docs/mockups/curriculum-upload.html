Add a curriculum to your library

One doc sets up the whole thing — your registration page, marketing flyer, parent emails, and instructor portal — without you re-typing a thing.

What this one doc sets up for you
  • A parent-facing registration listing — title, ages, description, themes, what kids will do
  • A marketing flyer + welcome emails you can send out to families
  • Session recap emails that go out to parents automatically after each class
  • An instructor portal with the full lesson plans, prep notes, and materials
  • A substitute handover so a sub can step in without scrambling
Your doc doesn't need to be perfect — if anything's missing (like the age range), we'll ask you a couple of quick questions on the next screen.
One thing before you upload: the curriculum title in your doc becomes the public name parents see — on your registration page, flyers, emails, all of it. Make sure it matches the class offering name you market. You'll get to edit on the next screen.

Behavior notes (for build)

  • Primary drop zone is the only required upload. Class/student materials zones only appear after the primary doc lands.
  • File limits: 25 MB each, up to 10 files per curriculum, allowed: .pdf .docx .xlsx .txt .md. Others rejected inline with a clear message.
  • Drive link: input is disabled in Chunk 2 (greyed out with "coming soon" hint). Chunk 2.5 turns it on.
  • On "Read my curriculum" click: create a curricula row at status='draft' with name = primary filename. Insert curriculum_documents rows for every uploaded file, tagged doc_type ∈ {primary, materials, journal}. Trigger the edge function on the primary doc. Route to /admin/curricula/:id/extracting.
  • Class/student materials are stored but not read by AI yet. They sit alongside the primary doc; we can have AI read them later if needed (no work for Jessica required).
  • Title-match rule (called out in the Plum-bordered note above): the title we read drives the public name parents see. Chunk 3 review screen will reinforce this with an explicit "doesn't match any of your existing programs — sure?" check before publishing.
  • Error states: bad file type, oversize, upload failure all surface inline above the CTA — never as a modal.
